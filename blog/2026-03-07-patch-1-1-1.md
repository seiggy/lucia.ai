**v1.1.1** is a patch release with targeted fixes and improvements.

<!-- truncate -->

## Brave Search — LLM Context API

The Brave Search plugin now uses the [Brave LLM Context API](https://api-dashboard.search.brave.com/documentation/services/llm-context), replacing the standard Web Search API. This delivers pre-extracted, LLM-ready web content to Lucia's agents in a single call — no scraping or URL fetching required.

## What's new

- **Pre-extracted content** — Agents receive page content directly from the search call, optimized for LLM consumption.
- **Token budget control** — Configure `MaxTokens` (1024–32768), `MaxUrls` (1–50), and `MaxSnippets` to fine-tune how much context reaches the agent.
- **Relevance filtering** — Adjustable `ThresholdMode` (`strict`, `balanced`, `lenient`, `disabled`) controls how aggressively the API filters for relevance.
- **Goggles support** — Use Brave's [Goggles](https://api-dashboard.search.brave.com/documentation/resources/goggles) to restrict or boost specific sources.
- **Location-aware queries** — Local and POI results when location headers are provided.

## Configuration

The plugin adds new configuration options manageable via the dashboard config GUI, environment variables, or `appsettings.json`:

| Key | Description | Default |
|---|---|---|
| `Plugins:BraveSearch:ApiKey` | Brave API subscription token (required) | — |
| `Plugins:BraveSearch:MaxTokens` | Approximate maximum tokens in context | `8192` |
| `Plugins:BraveSearch:MaxUrls` | Maximum URLs in the response | `20` |
| `Plugins:BraveSearch:ThresholdMode` | Relevance filtering mode | `balanced` |

## Upgrade notes

- **No breaking changes** — The plugin still registers `IWebSearchSkill`, so the GeneralAgent's `web_search` tool works without modification.
- **API key unchanged** — Your existing Brave Search API key works with the LLM Context endpoint.
- **Existing config preserved** — `BraveSearch:ApiKey` and `BRAVE_SEARCH_API_KEY` continue to work. New config keys (`MaxTokens`, `MaxUrls`, `ThresholdMode`) use sensible defaults if not set.

## Bug fixes

- **Agent definition domain settings persistence** — Domain settings now save correctly when updating an agent definition from the dashboard.
- **Plugin update detection** — The Plugin Management page now properly detects when a plugin update is available from its source repository and allows update/replacement directly from the UI.
